## はじめに

押し入れの奥底で眠っていたデバイスが、ふとした拍子に最新の開発環境と交差する瞬間があります。
今回取り上げるのは、かつて格安ノートPCとして話題になった「DGP-WNB1301」の再生活動です。
CPUは64bit（Atom Z3735F）でありながら、ブートローダー（UEFI）が32bit版しか受け付けないという、当時の格安機特有の「変態仕様」に阻まれ、多くのLinux導入試行が挫折してきました。
本記事では、Apple Silicon搭載のM1 MacBook Proという極めて現代的な環境から、この泥臭い32bitの壁を突破し、超軽量ディストリビューション antiX Linux を導入する手順を詳解します。

## 対象者

- 32bit UEFI（IA32）を搭載した低スペックPCの再利用に興味がある方
- M1 Mac環境でLinuxのライブUSBを安全に作成したい方
- メモリ2GBという極限のリソース管理を楽しめるエンジニア

## ハードウェアの特性と32bit UEFIの罠

DGP-WNB1301 のようなデバイスが抱える最大の問題は、ハードウェアの不一致にあります。
搭載されている Atom Z3735F 自体は 64bit 命令セットに対応していますが、マザーボード側のファームウェアが 32bit UEFI であるため、一般的な 64bit 用 Linux ISO をそのまま書き込んでも起動デバイスとして認識されません。
この「Mixed Mode」と呼ばれる構成において、OSインストーラーには `bootia32.efi` という 32bit 用のブートローダーが含まれている必要があります。

現代の主要なディストリビューションが 32bit サポートを打ち切る中、依然としてこの特殊な構成のブートに対応し、かつメモリ 2GB で実用性を維持できる選択肢は非常に限られています。
そこで浮上するのが、Debianベースで Systemd を採用しない設計思想を持つ、超軽量 Linux である antiX です。

## 開発環境としての M1 Mac とツールの選定

イメージの書き込みには、M1 MacBook Pro（macOS / ARM64）を使用します。
macOS の標準機能である `dd` コマンドでも同様の作業は可能ですが、書き込みミスの防止と比較的高速なベリファイ機能、そしてマウント解除の自動化を備えた balenaEtcher の Apple Silicon 版を採用するのが最も確実です。

antiX の公式ミラーサイトからは、必ず `i386`（32bit）版の ISO イメージをダウンロードしてください。
x64（64bit）版でも手動でブートローダーを追加すれば理論上は起動可能ですが、メモリ 2GB という制約を考慮すると、OS全体を 32bit 版で統一してメモリ使用量を抑えるのが合理的です。

## ライブUSBの作成とブート準備

ダウンロードした antiX の ISO イメージを balenaEtcher に読み込ませます。
対象の USB メモリを選択し、書き込みを開始します。
完了後、macOS から「このコンピュータで読み取れないディスクです」という警告が出ることがありますが、これは無視して取り出してください。

次に、ターゲットとなる DGP-WNB1301 側の設定変更が必要です。
電源投入直後に `F2` または `Del` キーを連打して BIOS（UEFI）設定画面に入り、以下の調整を行います。

1. Secure Boot を Disable に変更する
2. Boot Priority で、接続した USB メモリを最優先に移動させる

これらの設定を保存して再起動すると、antiX のブートメニューが表示されます。

## メモリ2GB環境でのインストールと最適化

インストーラー自体は非常に軽量ですが、物理メモリが 2GB しかないため、インストール中に不要なアプリケーションを開くのは避けるべきです。
インストール完了後、まず最初に行うべきは `zRAM` の調整です。

zRAM はメモリの一部を圧縮 RAM ディスクとして使用し、見かけ上のメモリ容量とスワップ速度を向上させる技術です。
antiX では `zram-tools` が標準で機能していますが、設定ファイル `/etc/default/zramswap` で圧縮アルゴリズムやサイズを微調整することで、ブラウジング時の安定性が向上します。

```bash
# /etc/default/zramswap の設定例
ALGO=zstd
PERCENT=50
```

また、2GB 環境での鬼門はウェブブラウザの選択です。
標準の Firefox でも動作はしますが、タブを 3 つ以上開くとスワップが発生して著しくレスポンスが低下します。
可能であれば、Badger などのさらに軽量なブラウザを常用するか、ターミナルベースのブラウザを活用することで、この小型機を「最高の SSH 端末」へと構造化できます。

## おわりに

私自身、最新の M1 Mac を使いながら 10 年前の「ゴミ」と呼ばれかねないハードウェアに命を吹き込む作業を通じて、いかにシステムをスリムに保つかというエンジニアとしての基本姿勢を問い直すことになりました。
32bit UEFI という特殊な崖を乗り越えた先に待っていたのは、かつてのネットブックが最新の Debian ベース OS で機敏に動くという、懐かしくも新しい景色でした。
皆さんも、引き出しで眠っている古いデバイスに、自分なりの構造を与えてみてはいかがでしょうか。
本記事が、かつての名機を再活用したいと考えている方の助けになれば幸いです。
