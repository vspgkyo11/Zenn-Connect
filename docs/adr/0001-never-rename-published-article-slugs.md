# 0001. 公開済み記事のファイル名（スラッグ）はリネームしない

- **Status**: Accepted
- **Date**: 2026-07-01

## Context

`articles/` に75本以上の記事が溜まり、`efd31b50df3172.md` のような不規則な文字列のファイル名では内容が把握できず、「見やすい名前にリネームしたい」という要望が出た。

Zennの仕様として、**ファイル名（スラッグ）＝記事ID＝公開URL** であり、一度公開した記事のスラッグは**仕様上変更不可**である（`articles/xxx.md` → `https://zenn.dev/{user}/articles/xxx`）。UI上・API上ともに「後からスラッグだけ変える」手段は提供されておらず、ファイルをリネームする＝そのスラッグの記事を削除して別記事を新規作成するのと同義になる。スラッグの文字種（`a-z`・`0-9`・`-`・`_`）や長さ（12〜50文字）も固定の技術的制約であり、公開前にしか選び直せない。

## Decision

上記の仕様上の制約を前提に、`published: true` の記事のファイル名は**一切リネームしない**。

見やすさの問題は、ファイル名を変えるのではなく別レイヤーで解決する。

- 人間向けの目次: `docs/article_index.md`（`scripts/generate_index.py` で自動生成）
- 新規記事のみ: `npx zenn new:article --slug <カテゴリ>-<トピック>` で意味のあるスラッグを最初から指定する（詳細は [docs/repository_guide.md](../repository_guide.md) §4）

## Consequences

- 既存75本のスラッグは今後も不規則な文字列のまま残る（許容する）
- 記事を追加・更新したら `docs/article_index.md` の再生成を忘れると索引が陳腐化するので、運用上の注意点として残る
- `published: false` の下書きはURLが未確定なので、公開前であればスラッグ変更の余地がある
